# PII Obfuscator Project

## Обзор проекта

Этот проект представляет собой решение для обфускации (маскирования) конфиденциальных данных (PII - Personally Identifiable Information) в текстовых документах с использованием моделей Named Entity Recognition (NER) из библиотеки Hugging Face Transformers. Проект демонстрирует полный цикл fine-tuning предобученной multilingual NER модели для адаптации к распознаванию PII на русском языке и последующего использования этой модели для обфускации.

Проект разработан с учетом принципов модульности и разделения ответственности, что делает его легко расширяемым и поддерживаемым.

## Функционал

*   **Fine-tuning NER модели:** Реализовал две стратегии дообучения (fine-tuning) BERT-based модели для задачи распознавания именованных сущностей (NER):
       * Full Fine-tuning для достижения максимальной точности.
       * Parameter-Efficient Fine-Tuning (PEFT) с использованием LoRA, что позволило значительно сократить требования к вычислительным ресурсам и время обучения.
*   **Обфускация текста:** Замена распознанных PII-сущностей на анонимные маркеры (например, "person-1", "location-2").
*   **Гибкая конфигурация:** Все ключевые параметры проекта вынесены в отдельный файл конфигурации.
*   **CLI интерфейс:** Разработал интерфейс командной строки (CLI) на argparse для удобного запуска процессов обучения и инференса, что позволяет гибко управлять поведением системы через параметры.

### 2. Установка зависимостей

```bash
pip install torch torchvision torchaudio transformers datasets evaluate seqeval numpy scikit-learn
```
(Примечание: `scikit-learn` добавлена для `train_test_split`).

### 3. Конфигурация проекта

Все основные настройки находятся в файле `src/config.py`:
*   `MODEL_NAME`: Имя базовой модели из Hugging Face Hub (например, "Babelscape/wikineural-multilingual-ner").
*   `LABELS_LIST`: Полный список меток, которые должна предсказывать модель (включая BIO-метки).
*   `OUTPUT_DIR`: Путь для сохранения промежуточных чекпоинтов обучения.
*   `FINETUNED_MODEL_PATH`: Путь для сохранения финальной дообученной модели.
*   `TRAINING_DATA_PATH`: Путь к JSON-файлу с обучающими данными.

Вы можете изменять эти параметры в соответствии с вашими потребностями.

## Использование CLI

Все команды запускаются из корневой директории проекта ai_portfolio/pii_obfuscator/ и имеют следующий вид:

   1 python -m src.main_cli [КОМАНДА] [ФЛАГИ]

  Существует две основные команды: train и obfuscate.

  ---

  1. Команда train (Обучение модели)

  Эта команда используется для дообучения модели на ваших данных.

  Флаги для train:

   * --strategy [full|lora]
       * Назначение: Указывает, какую стратегию дообучения использовать.
       * Значения:
           * full — полноценное дообучение всех слоев модели. Создает полную копию модели в models/my_finetuned_ner_model.
           * lora — эффективное дообучение только LoRA-адаптеров. Создает небольшие файлы адаптеров в models/my_lora_adapters.
       * По умолчанию: full.

  Примеры использования train:

   * Обучение по умолчанию (полное дообучение):
   1     python -m src.main_cli train
      (эквивалентно `python -m src.main_cli train --strategy full`)

   * Обучение с использованием LoRA:

   1     python -m src.main_cli train --strategy lora

  ---

  2. Команда obfuscate (Обфускация текста)

  Эта команда используется для маскирования PII в тексте. Она имеет больше флагов для гибкого выбора модели.

  Флаги для obfuscate:

   * --text "..."
       * Назначение: Обязательный флаг, в котором передается текст для обфускации.
   * --use_finetuned [true|false]
       * Назначение: Указывает, использовать ли локально дообученную модель или базовую модель из Hugging Face.
       * Значения:
           * true — использовать дообученную модель (какую именно — определяется флагом --strategy).
           * false — использовать базовую модель Babelscape/wikineural-multilingual-ner напрямую, без каких-либо доработок.
       * По умолчанию: true.
   * --strategy [full|lora]
       * Назначение: Этот флаг имеет значение, только если `--use_finetuned true`. Он указывает, какую из дообученных моделей загружать.
       * Значения:
           * full — загрузить модель из models/my_finetuned_ner_model.
           * lora — загрузить базовую модель и применить к ней адаптеры из models/my_lora_adapters.
       * По умолчанию: full.

  Примеры использования obfuscate:

   * Обфускация с использованием дообученной модели (по умолчанию):

   1     python -m src.main_cli obfuscate --text "Иван Петров живет в Москве."
      (Загрузит модель после полного дообучения из `models/my_finetuned_ner_model`)

   * Обфускация с использованием LoRA-адаптеров:

   1     python -m src.main_cli obfuscate --text "Текст для модели с LoRA." --strategy lora
      (Здесь `--use_finetuned` по умолчанию `true`, а `--strategy` явно указывает на `lora`)

   * Обфускация с использованием базовой модели (без дообучения):
   1     python -m src.main_cli obfuscate --text "Этот текст обработает базовая модель." --use_finetuned false
      (Флаг `--strategy` в этом случае будет проигнорирован)

## Расширение функционала

### Добавление новых типов PII (например, телефонные номера)

1.  **Обновите `src/config.py`:** Добавьте новые метки (например, `B-PHONE`, `I-PHONE`) в `LABELS_LIST`.
    ```python
    LABELS_LIST = ['O', 'B-PER', 'I-PER', ..., 'B-PHONE', 'I-PHONE']
    ```
2.  **Обновите `data/training_data.json`:** Создайте новые обучающие примеры, содержащие телефонные номера, и разметьте их соответствующими метками.
    ```json
    {
        "words": ["Мой", "номер", "телефона", "+79001234567", "."],
        "labels": ["O", "O", "O", "B-PHONE", "O"]
    }
    ```
3.  **Обновите `PIIObfuscator.label_map` в `src/inference_engine.py`:** Добавьте новую метку в ваш переводчик, например:
    ```python
    self.label_map = {
        # ... существующие метки ...
        "PHONE": "phone",
    }
    ```
4.  **Переобучите модель:** Запустите `python -m src.main_cli train`.
5.  **Протестируйте обфускацию:** Используйте команду `obfuscate` с текстом, содержащим телефонные номера.


## Паттерны проектирования

В проекте используются следующие паттерны и принципы:

*   **Модульность и Разделение Ответственности (Modularity & Separation of Concerns):** Каждый файл (`config.py`, `data_processor.py` и т.д.) отвечает за свою четко определенную область.
*   **Стратегия (Strategy Pattern, неявно):** Класс `PIIObfuscator` принимает `ner_callable` (фактически, пайплайн Transformers), что позволяет легко менять "стратегию" распознавания сущностей.
*   **Фабричный метод (Factory Method Pattern, неявно):** Функция `create_inference_pipeline` выступает как фабрика для создания объектов `PIIObfuscator`.
*   **Объект конфигурации (Configuration Object Pattern):** `config.py` централизует и предоставляет доступ ко всем параметрам проекта.

## Тестирование

Проект покрыт набором автоматических тестов для обеспечения корректности и стабильности работы. Тесты написаны с использованием фреймворка `pytest` и находятся в директории `tests/`.

Для запуска всех тестов выполните следующую команду из корневой директории проекта (`pii_obfuscator_project`):

```bash
pytest
```

Тесты используют фикстуры из `tests/conftest.py` для создания изолированного окружения, что гарантирует отсутствие влияния на реальные данные и модели.

### Что проверяют тесты:

*   **`tests/test_config.py`**:
    *   Проверяет, что все переменные в файле конфигурации `src/config.py` определены и имеют правильные типы.
    *   Убеждается, что пути к данным и моделям строятся корректно.

*   **`tests/test_data_processor.py`**:
    *   Тестирует логику загрузки и обработки данных из `src/data_processor.py`.
    *   Проверяет, что токенизатор загружается правильно.
    *   Валидирует, что функция `tokenize_and_align_labels` корректно преобразует слова и метки в формат, пригодный для модели.

*   **`tests/test_inference_engine.py`**:
    *   Тестирует ядро логики обфускации из `src/inference_engine.py`.
    *   Проверяет, что класс `PIIObfuscator` правильно заменяет найденные сущности на анонимные маркеры.
    *   Тестирует функцию `create_inference_pipeline`, проверяя, что она корректно загружает нужную модель в зависимости от настроек: базовую, полностью дообученную или модель с LoRA-адаптерами. (Важно: реальные модели при этом не загружаются, используется мокинг).

*   **`tests/test_main_cli.py`**:
    *   Тестирует интерфейс командной строки `src/main_cli.py`.
    *   Проверяет, что команды `train` и `obfuscate` с различными аргументами (`--strategy`, `--use_finetuned`) работают как ожидается и вызывают соответствующие функции.
