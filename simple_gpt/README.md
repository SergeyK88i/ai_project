# SimpleGPT: Реализация GPT-архитектуры с нуля

## 1. Обзор проекта

Этот проект представляет собой реализацию с нуля компактной модели-трансформера (GPT) для генерации текста на уровне символов (character-level). Цель проекта — продемонстрировать понимание архитектуры GPT и всех ключевых компонентов, необходимых для ее обучения и работы, без использования высокоуровневых библиотек, таких как `transformers.Trainer`.

Весь код, включая модель, токенизатор, датасет и цикл обучения, находится в одном файле `mini_language_model.py` и снабжен подробными комментариями.

## 2. Архитектура и компоненты

Проект разбит на несколько логических классов, каждый из которых выполняет свою задачу:

*   **`CharTokenizer`**: Простой, но полнофункциональный токенизатор, который создает словарь из уникальных символов в обучающем тексте и выполняет преобразования `string <-> list[int]`.

*   **`ShakespeareDataset`**: Кастомный класс датасета для PyTorch (`torch.utils.data.Dataset`), который принимает на вход сырой текст и на лету формирует обучающие пары "контекст-цель" (`x`, `y`) необходимой длины (`block_size`).

*   **`SimpleGPT`**: Основной класс модели, реализующий архитектуру decoder-only трансформера. Включает в себя:
    *   **Слои эмбеддингов:** Токеновые (`wte`) и позиционные (`wpe`) эмбеддинги.
    *   **Блоки трансформера (`Block`):** Состоят из:
        *   **`CausalSelfAttention`**: Реализация механизма "причинного" внимания, которое "смотрит" только на предыдущие токены в последовательности. Для повышения эффективности используется встроенная в PyTorch 2.0+ функция `F.scaled_dot_product_attention` (Flash Attention).
        *   **`MLP`**: Стандартный двухслойный перцептрон с функцией активации GELU.
        *   **`LayerNorm`** и остаточные соединения (residual connections).
    *   **`lm_head`**: Финальный линейный слой для предсказания следующего токена.

*   **`Trainer`**: Класс, инкапсулирующий логику цикла обучения. Он управляет получением батчей, выполнением прямого и обратного проходов (forward/backward pass) и шагами оптимизатора.

## 3. Паттерны проектирования

В коде явно или неявно применены следующие паттерны:

*   **Configuration Object:** Все гиперпараметры модели (размер словаря, количество слоев, голов внимания и т.д.) вынесены в датакласс `GPTConfig`, что упрощает эксперименты.
*   **Composition:** Сложные модули, такие как `Block` и `SimpleGPT`, "собираются" из более простых, переиспользуемых компонентов (`CausalSelfAttention`, `MLP`).
*   **Facade / Strategy:** Класс `Trainer` выступает "фасадом", скрывая сложную логику цикла обучения за простым методом `train()`. Он также реализует "стратегию" обучения.
*   **Adapter / Iterator:** `ShakespeareDataset` адаптирует сырые текстовые данные к формату, ожидаемому PyTorch, и предоставляет итератор для доступа к обучающим примерам.

## 5. Установка и запуск

### 5.1. Установка зависимостей

Для работы скрипта необходима библиотека `torch`. Установите ее, если она отсутствует:
```bash
pip install torch
```

### 5.2. Запуск

Проект является самодостаточным исполняемым скриптом. Для запуска демонстрации (обучение и генерация текста) выполните команду, находясь в корневой директории `ai_portfolio`:

```bash
python simple_gpt/mini_language_model.py
```

### Что ожидать в выводе:

1.  **Демонстрация токенизатора:** Будет показан размер словаря и пример кодирования/декодирования.
2.  **Демонстрация датасета:** Будет выведен первый обучающий пример (входная и целевая последовательности).
3.  **Запуск обучения:** Начнется итеративный процесс обучения модели, в консоль будут выводиться текущая ошибка (Loss).
4.  **Генерация текста:** После обучения модель сгенерирует и выведет в консоль новый текст в стиле А.С. Пушкина, начиная с новой строки.
